브라우저 런타임 구조:
- KpiRuntime은 섹션별 기능을 느슨하게 연결하는 전역 레지스트리입니다.
- registerSectionRenderer: 섹션 화면 렌더러 등록
- registerSectionInitializer: 섹션 진입 후 초기화 작업 등록
- registerDefaultContentDecorator: 기본 콘텐츠 보조 렌더링 등록
- registerShortcutCloser: 팝업/단축 레이어 닫기 등록
- registerBootTask: 앱 부팅 시 실행할 작업 등록
- registerDefaultOpenOptions: 섹션 기본 열기 옵션 등록
- registerViewChangeHandler: 화면 전환 후 처리 등록
- registerSaveProvider: 전역 저장 버튼에서 사용할 저장 동작 등록
- resolveSaveAction: 현재 화면에서 우선순위가 가장 높은 저장 동작 선택
- recordSaveHistory: 저장 기록 저장
- getServerRuntimeConfig/hasServerAuthority/canUseServerWrite: 서버 모드와 쓰기 가능 여부 판단

답변 구성 예시:
1. 문제 인식:
   제조 운영 업무는 생산량, 유틸리티 사용량, 점검 기록, 작업 이력이 분리되면 현황 파악과 증빙 관리가 어렵다.
2. 설계:
   업무 흐름을 팀 캘린더, 유틸리티 입력, 생산량 추출, 오디트, 설비 이력 카드로 나누고 공통 대시보드 셸과 섹션 런타임으로 연결했다.
3. 구현:
   JavaScript 기반 정적 데모와 Node/PostgreSQL 선택형 런타임을 구성하고 입력/분석/저장/문서/운영 콘솔을 모듈화했다.
4. 검증:
   targeted smoke test, Playwright, lint, format check, 브라우저 확인, 민감 문자열 스캔으로 공개 안전성과 동작을 점검했다.
5. 배운 점:
   업무 도구는 화면을 만드는 것뿐 아니라 데이터 흐름, 예외 처리, 저장 책임, 검증 가능성을 함께 설계해야 오래 유지된다.
